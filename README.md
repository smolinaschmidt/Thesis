# Remaking Color

Thesis platform for studying how color language changes across movie remakes over time.

## Stack
- Website: static HTML + CSS + vanilla JS modules (`index.html`, `styles.css`, `js/*.js`). D3 is loaded as an ES module from a CDN. No build step.
- Data pipeline: Python scripts (`scripts/`).
- Source dataset: `data/Final_Database.csv`.

## Project structure
- `index.html`: single page that hosts the scroll narrative.
- `styles.css`: all UI styles.
- `js/main.js`: entry — loads JSON, runs the scroll observer, wires the search.
- `js/color.js`, `js/tooltip.js`: shared helpers.
- `js/intro.js`, `js/timeline.js`, `js/method.js`, `js/morph.js`, `js/case-study.js`, `js/conclusion.js`: one module per scrolly section.
- `scripts/*.py`: Python pipeline to clean/enrich/analyze the dataset.
- `data/`: all collected data.
- `data/analisis/`: analysis outputs generated by Python scripts.

## Quick start
1. Build processed data:
   - `python3 scripts/build_remake_families.py`
   - `TMDB_API_KEY=<your_key> python3 scripts/enrich_tmdb_metadata.py`
   - `python3 scripts/build_sentiment_features.py`
   - `python3 scripts/generate_color_analytics.py`
   - `python3 scripts/data_quality_report.py`
2. Open the website locally with any static server:
   - `npm start`  (runs `python3 -m http.server 5173`)
   - or `python3 -m http.server 5173`
   - Open `http://localhost:5173/`
3. Optional: enable live TMDB API enrichment:
   - Copy `config.example.js` to `config.js`
   - Add `TMDB_API_KEY` in `config.js`
